# Spec-First Law

Before writing any non-trivial code:

1. A `SPEC.md` must exist for the task.
2. The SPEC must include: intent, acceptance criteria, out-of-scope items,
   and a rough plan.
3. The user must acknowledge the SPEC (explicitly or by saying "go").

If a SPEC does not exist, invoke `/spec` or write one. Do not skip this.

Exceptions: trivial fixes (typo, obvious one-line bug) may proceed without a
SPEC but must still be journaled.
